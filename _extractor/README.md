# Экстрактор + оценка точности (главная часть)

Вытаскивает данные из PDF/PNG израильских документов в структурный JSON и МЕРЯЕТ точность
против ground-truth. Обобщается на новые варианты вёрстки (vision-LLM), без кода под шаблон.

## Установка / запуск (нужна СЕТЬ + ключ)
    pip install anthropic pdf2image pillow      # + системный poppler (pdftoppm) для PDF
    export ANTHROPIC_API_KEY=...
    # один документ:
    python3 extract.py путь/документ.png > out.json
    # весь датасет + отчёт точности:
    python3 run_all.py --dataset .. --pred pred

## Файлы
- extract.py  — PDF/PNG -> JSON. Vision + tool-use (схема EXTRACTION_TOOL форсирует структуру).
                Модель: env ANTHROPIC_MODEL (default claude-opus-4-8). `--dry-run` собирает
                запрос без вызова API (оффлайн-проверка пломбинга).
- evaluate.py — сверка предсказаний с ground-truth ПО ПОЛЯМ. Идентификаторы (id, номера,
                מספר הקצאה) — строгий матч; денежные поля — допуск 0.5%.
- run_all.py  — батч: extract по всем PNG -> pred/<id>.json -> отчёт точности.
- sample_eval_report.txt — образец отчёта (сгенерирован на искусственно «зашумлённых»
                предсказаниях, чтобы показать, что харнесс различает ошибки).

## Проверено здесь (без сети)
- Сборка запроса (--dry-run) и кодировка PNG — ок.
- Парсинг tool-ответа — ок (на стабе).
- Оценка: идеальные предсказания -> 100.0% (741 поле, 24 док.);
  зашумлённые -> 94.9%, слабые поля подсвечены (customer.id 0%, allocation_number 47%).
Сам вызов API здесь не гонялся — в песочнице нет сети. Запускается у тебя с ключом.

## Портирование на OpenAI/Gemini
Тот же паттерн: картинка (base64) + системный промпт + инструмент/JSON-schema.
Меняется только клиент и формат сообщения; EXTRACTION_TOOL (схема) остаётся тот же.
